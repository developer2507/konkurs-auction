# Исправление ошибок импорта TypeScript

## Проблема: "Cannot find module 'mongoose'" и подобные ошибки

Эти ошибки появляются потому, что:
1. Не установлены зависимости Node.js
2. TypeScript не может найти типы

## Решение по шагам

### Шаг 1: Установите все зависимости

Откройте терминал в папке `C:\Konkurs` и выполните:

```bash
npm install
```

Эта команда установит все пакеты из `package.json`, включая:
- mongoose
- express
- redis
- и все остальные зависимости

**Процесс может занять 2-5 минут.**

### Шаг 2: Проверьте, что зависимости установлены

Проверьте, что папка `node_modules` создана:

```bash
dir node_modules
```

Должна быть папка с множеством подпапок.

### Шаг 3: Перезапустите редактор кода

Если вы используете VS Code или другой редактор:
1. Закройте редактор полностью
2. Откройте заново
3. Дождитесь, пока TypeScript загрузит типы (может занять минуту)

### Шаг 4: Проверьте, что типы установлены

Убедитесь, что установлены типы для Node.js:

```bash
npm list @types/node
```

Если команда выводит ошибку, установите вручную:

```bash
npm install --save-dev @types/node
```

### Шаг 5: Попробуйте собрать проект

```bash
npm run build
```

Если сборка успешна - ошибки в редакторе должны исчезнуть после перезапуска.

## Если ошибки остаются

### Вариант 1: Очистка и переустановка

```bash
# Удалите папку node_modules и файл package-lock.json
rmdir /s /q node_modules
del package-lock.json

# Переустановите зависимости
npm install
```

### Вариант 2: Обновите npm

```bash
npm install -g npm@latest
npm install
```

### Вариант 3: Проверьте версию Node.js

```bash
node --version
```

Должна быть версия 18+ (лучше 20+).

Если версия старая:
1. Скачайте Node.js 20 LTS: https://nodejs.org/
2. Переустановите
3. Перезапустите терминал
4. Выполните `npm install` снова

## Типичные ошибки и решения

### Ошибка: "Cannot find module 'dotenv'"

**Решение:**
```bash
npm install dotenv
```

### Ошибка: "Cannot find module 'express'"

**Решение:**
```bash
npm install express @types/express
```

### Ошибка: "Cannot find name 'process'"

**Решение:**
```bash
npm install --save-dev @types/node
```

### Ошибка: "Module not found: Can't resolve 'socket.io-client'"

Это ошибка frontend. Решение:
```bash
cd frontend
npm install
cd ..
```

## Проверка правильности установки

Выполните следующие команды и проверьте, что нет ошибок:

```bash
# Проверка версии Node.js
node --version

# Проверка версии npm
npm --version

# Проверка установленных пакетов
npm list --depth=0

# Попытка собрать проект
npm run build
```

Если все команды выполняются без ошибок - всё установлено правильно!

## Что делать, если ничего не помогает

1. **Убедитесь, что вы в правильной папке:**
   ```bash
   cd C:\Konkurs
   ```

2. **Проверьте package.json:**
   ```bash
   type package.json
   ```
   Убедитесь, что файл содержит все зависимости.

3. **Попробуйте другой терминал:**
   - Используйте PowerShell от имени администратора
   - Или используйте Git Bash
   - Или используйте обычный CMD

4. **Очистите кэш npm:**
   ```bash
   npm cache clean --force
   npm install
   ```

5. **Проверьте интернет-соединение:**
   - npm нужно скачивать пакеты из интернета
   - Если есть прокси, настройте его в npm

## После успешной установки

После того, как `npm install` завершится успешно:

1. Создайте файл `.env` (см. CREATE_ENV.md)
2. Запустите Docker контейнеры
3. Выполните `npm run build`
4. Запустите `npm run dev`

Всё должно работать!
